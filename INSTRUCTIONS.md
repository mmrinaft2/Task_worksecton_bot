# Odoo Task Management Bot — Інструкція

## Що це?

Telegram-бот, який допомагає створювати технічні завдання (ТЗ) для розробників Odoo і автоматично відправляє їх у Worksection.

Бот працює як **віртуальний Odoo-архітектор**: аналізує опис задачі, задає уточнюючі питання, генерує повне ТЗ і створює задачу в проєктній системі.

---

## Можливості бота

### 1. Створення задачі з опису
- Користувач описує проблему або ідею **текстом** або **голосовим повідомленням**
- AI автоматично визначає категорію задачі: `bug`, `feature`, `improvement`, `support`
- AI генерує 2–3 уточнюючих питання, специфічних для конкретної задачі
- Після відповідей AI складає повне технічне завдання

### 2. Голосові повідомлення
- Бот приймає голосові повідомлення
- Автоматично транскрибує через OpenAI Whisper
- Показує розпізнаний текст і обробляє його як звичайне повідомлення

### 3. Прикріплення файлів
- Підтримка: документи, фото, відео
- Файли прикріплюються до задачі в Worksection
- Файли більше 20 МБ: бот пропонує завантажити на Google Drive і надіслати посилання

### 4. Вибір дедлайну
- Інтерактивний календар для вибору терміну виконання
- Можливість пропустити (без дедлайну)
- Минулі дати заблоковані

### 5. AI-генерація ТЗ
- Бот генерує структуроване ТЗ з розділами:
  - **Назва** — коротка назва задачі
  - **Мета** — яку бізнес-проблему вирішує
  - **Технічний опис** — що потрібно зробити в Odoo (моделі, поля, логіка)
  - **Кроки реалізації** — покрокова інструкція
  - **Критерії прийняття** — що має працювати після завершення

### 6. Підтвердження та редагування
- Після генерації ТЗ користувач бачить три кнопки:
  - **Так, все правильно** — відправити задачу в Worksection
  - **Редагувати** — описати задачу заново (прикріплені файли зберігаються)
  - **Скасувати** — відмінити створення

### 7. Інтеграція з Worksection
- Задача автоматично створюється у Worksection з:
  - Назвою (згенерованою AI)
  - Текстом (опис + відповіді на питання + посилання)
  - Прикріпленими файлами
  - Пріоритетом та дедлайном
- Бот повертає посилання на створену задачу

### 8. Перегляд задач (Мої задачі)
- Показує список усіх задач проєкту з Worksection
- Біля кожної задачі відображається **цифра пріоритету** — так само як у Worksection:

| Цифра | Значення | Опис |
|:---:|---|---|
| **10** | Блокер | Блокує роботу інших задач, потребує негайного вирішення |
| **9** | Критичний | Серйозна проблема, яку потрібно вирішити якнайшвидше |
| **8** | Терміново | Висока терміновість, потрібно зробити найближчим часом |
| **7** | Високий | Важлива задача з високим пріоритетом |
| **6** | Вище середнього | Трохи важливіша за звичайні задачі |
| **5** | Нормальний | Стандартний пріоритет за замовчуванням |
| **4** | Середній | Звичайна задача без терміновості |
| **3** | Нижче середнього | Може почекати |
| **2** | Низький | Виконати, коли буде час |
| **1** | Мінімальний | Найнижчий пріоритет |
| ✅ | Виконана | Задача завершена |

- Задачі відсортовані: найкритичніші (10) зверху, виконані (✅) — в кінці
- Натисніть на задачу, щоб побачити деталі: статус, пріоритет, дедлайн, автор, виконавець та посилання на Worksection
- Список розбитий на сторінки (по 8 задач), навігація кнопками ◀ ▶

### 9. Головне меню
- Через 10 хвилин після створення або скасування задачі автоматично з'являється головне меню
- Кнопки: **"Створити задачу"** та **"Мої задачі"**
- Якщо користувач починає писати — запланований показ меню скасовується

### 10. Багатомовність
Підтримка 4 мов (визначається при старті):
- 🇺🇦 Українська (за замовчуванням)
- 🇬🇧 English
- 🇷🇺 Русский
- 🇵🇱 Polski

## Як користуватися

### Крок 1: Запуск
Надішли команду `/start` у Telegram-бот.

### Крок 2: Створення задачі
Натисни **"Створити задачу"** або просто надішли текст з описом проблеми/ідеї.

> Приклад: "При створенні інвойсу в модулі Accounting не підтягується банківський рахунок контрагента"

### Крок 3: Відповіді на питання
Бот задасть 2–3 уточнюючих питання. Відповідай або напиши **"пропустити"** для переходу до наступного.

### Крок 4: Вибір дедлайну
Обери дату в календарі або натисни **"Пропустити"**.

### Крок 5: Перевірка ТЗ
Прочитай згенероване ТЗ та натисни:
- ✅ **Так, все правильно** — задача створиться у Worksection
- ✏️ **Редагувати** — опиши задачу заново
- ❌ **Скасувати** — відмінити

### Додатково: файли
На будь-якому етапі (до підтвердження ТЗ) можна надіслати:
- 📄 Документи (PDF, XLSX, DOCX тощо)
- 📸 Скріншоти / фото
- 🎥 Відео (до 20 МБ, або через Google Drive)
- 🔗 Посилання на Google Drive для великих файлів

### Додатково: голос
Замість тексту можна надіслати голосове повідомлення — бот розпізнає його і обробить як текст.

## Технічні вимоги

### Змінні середовища (`.env`)
```
TELEGRAM_BOT_TOKEN=...        # Токен Telegram-бота
OPENAI_API_KEY=...            # API-ключ OpenAI (GPT + Whisper)
WORKSECTION_API_TOKEN=...     # API-ключ Worksection
WS_ACCOUNT_DOMAIN=...         # Домен Worksection (наприклад: company.worksection.com)
WORKSECTION_PROJECT_ID=...    # ID проєкту в Worksection
```

### Залежності
```
python-telegram-bot==20.7
python-dotenv
openai
requests
langdetect
```

### Запуск
```bash
pip install -r requirements.txt
python main.py
```

## Архітектура

| Файл | Опис |
|---|---|
| `main.py` | Основна логіка бота: хендлери, меню, календар, переклади |
| `ai_agent.py` | AI-аналіз задач (категорія + питання) та генерація ТЗ через OpenAI |
| `worksection_api.py` | Інтеграція з Worksection API (створення задач, хешування) |
| `.env` | Конфігурація (токени, ключі) |
